# 成绩分析统计功能设计

## Context

成绩汇总页面（`ClassReportView.vue`）当前只有班级+实验筛选和成绩表格，缺少分析统计能力。教师需要直观了解成绩分布情况，以便调整教学策略。

## 改动范围

仅修改前端页面 `labex-ui/src/views/teacher/ClassReportView.vue`，新增 ECharts 依赖。不涉及后端改动，所有统计由前端根据现有接口返回的成绩数据计算。

## 页面布局

查询后页面分为三个区域，从上到下：

### 1. 筛选条件（保持不变）

班级下拉 + 实验下拉 + 查询按钮

### 2. 左右分栏区域（查询后显示）

**左侧 — 4 张统计卡片**（纵向排列，固定宽度 280px）：

| 卡片 | 计算方式 | 配色 |
|------|---------|------|
| 平均分 | 所有学生分数求平均 | 绿色边框 |
| 最高分 | `Math.max` | 蓝色边框 |
| 最低分 | `Math.min` | 橙色边框 |
| 及格率 | 分数 >= 60 的人数 / 总人数 × 100% | 红色边框 |

**右侧 — 两个图表**（纵向排列）：

- **柱状图**：5 个分数段（0-59、60-69、70-79、80-89、90-100）的人数分布
- **饼图**：同 5 个分数段的人数占比

### 3. 成绩明细表格（保持不变）

学号、姓名、班级、总分

## 技术选型

- 图表库：**ECharts**，通过 `npm install echarts` 安装
- 统计计算：前端 JS 根据 `scores` 数组实时计算，无需后端改动

## 数据流

```
用户选择班级+实验 → 调用 api.getClassScore() → 返回 scores 数组
  → 前端计算统计指标（平均分/最高分/最低分/及格率）
  → 前端按分数段分组统计人数
  → 渲染卡片 + 柱状图 + 饼图
```

## 关键文件

- `labex-ui/src/views/teacher/ClassReportView.vue` — 唯一需要修改的文件
- `labex-ui/package.json` — 新增 echarts 依赖

## 验证方式

1. 选择一个有成绩数据的班级和实验，点击查询
2. 确认 4 张卡片数值正确（可用 Excel 交叉验证）
3. 确认柱状图和饼图的分数段人数加总等于学生总数
4. 选择一个无成绩数据的实验，确认统计和图表区域不显示
